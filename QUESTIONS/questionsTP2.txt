2.1 What are testcontainers?
Testcontainers is a Java testing library that allows you to run real services
(like PostgreSQL or MySQL) inside Docker containers during tests.
It creates and removes these containers automatically
giving a clean and realistic environment for each test run.

2-2 For what purpose do we need to use secured variables ?

Secured variables are used to store sensitive data like API keys, passwords, or tokens.
They are hidden from the code and logs to avoid leaks.
This improves security in CI/CD pipelines. They also allow safe access control across environments.

2-3 Why did we put needs: build-and-test-backend on this job? Maybe try without this and you will see!

We use needs to make one job depend on another. 
Here, it ensures this job runs only if the backend build and tests succeed.
Without it, jobs may run in parallel. This could allow later steps to run even if tests fail, which is unsafe.

2-4 — Purpose of pushing Docker images

We push Docker images to store them in a remote registry like Docker Hub.
This makes the images accessible from anywhere, not just the CI runner.
It allows deployment systems (servers, Kubernetes, etc.) to pull and run the same built version.
It also ensures version consistency between environments.
Pushing images is essential for CI/CD pipelines, because it separates the build stage from the deployment stage.
Once an image is pushed, it becomes a reusable artifact that can be deployed multiple times without rebuilding.
